# 经验教训

- 记录值得长期保留的项目经验与回归教训。
- 写入前优先确认它是否应先经过 reflection inbox 审核。

## 使用规则

- 只记录跨任务、可复用、会改变后续执行方式的规则。
- 每条规则都应保留来源 reflection 或稳定文档路径。
- 不把原始问题叙事、一次性排障记录或未审候选直接抄进这里。
- 当 `inbox/reflections/` 的 proposed 条目超过 50 时，优先更新本文件，再归档已完成提炼的 reflection。
